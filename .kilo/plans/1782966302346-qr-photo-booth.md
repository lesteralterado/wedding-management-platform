# QR Photo Booth & Shared Wedding Gallery - Implementation Plan

## Overview
Implement a QR Photo Booth feature allowing wedding guests to capture/upload photos/videos via browser camera without app installation.

---

## Decisions

### 1. Database Model Location
**Decision**: Add `Media` model to `prisma/schema.prisma`

```prisma
model Media {
  id              String    @id @default(cuid())
  weddingId       String
  wedding         Wedding   @relation(fields: [weddingId], references: [id], onDelete: Cascade)
  uploadedByGuestId String?
  uploadedByName  String?
  type            MediaType @default(PHOTO)
  url             String
  thumbnail       String?
  status          MediaStatus @default(PENDING)
  uploadedAt      DateTime  @default(now())
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt

  @@index([weddingId])
  @@index([status])
}

enum MediaType {
  PHOTO
  VIDEO
}

enum MediaStatus {
  PENDING
  APPROVED
}
```

**Reason**: Separate model enables status tracking and independent ownership.

### 2. Wedding Settings
**Decision**: Extend `Wedding` model with photo booth settings:
- `photoBoothEnabled Boolean @default(false)` - Toggle to enable/disable
- `photoBoothAutoApprove Boolean @default(false)` - Auto-publish vs manual approval

### 3. Routing Structure
**Decision**:
- Public photo booth: `/(public-wedding)/weddings/[weddingSlug]/photo-booth/page.tsx`
- Dashboard: `/app/dashboard/photo-booth/page.tsx`
- API: `/app/api/photo-booth/route.ts`

### 4. Camera Implementation
**Decision**: Client component using browser MediaDevices API (`getUserMedia`) with:
- Live camera preview via `<video>` element
- Front/back camera switching
- Photo capture via `<canvas>`
- Optional video recording (future - for now photo only)
- Countdown timer (3s, 5s, 10s options)

### 5. Storage
**Decision**: Extend existing `src/lib/upload/upload.ts` to save media to `public/uploads/photobooth/`

### 6. Guest Identity
**Decision**: Guests enter name before uploading (no auth required). Name stored in `uploadedByName`.

- Large uploads may timeout on slow connections
